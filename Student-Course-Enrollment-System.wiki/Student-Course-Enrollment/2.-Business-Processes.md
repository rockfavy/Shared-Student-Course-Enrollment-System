# 2. User Stories & Processes

## 2.1 Student Registration & Login

**User Story:** As a student, I want to register and log into the system so that I can securely access my course information.

**Process Flow:**
1. Student navigates to registration page  
2. Student enters registration details (email, password, name)  
3. System validates input  
4. System creates student account  
5. System returns authentication token  
6. Student is redirected to courses page  

**Login Process:**
1. Student navigates to login page  
2. Student enters credentials  
3. System validates credentials  
4. System returns authentication token  
5. Student is granted access  

---

## 2.2 View Available Courses

**User Story:** As a student, I want to see a list of available courses so I can select which to enroll in.

**Process Flow:**
1. Student navigates to courses page  
2. System retrieves list of courses from API  
3. System displays courses with details  
4. Student can view course information  
5. Student can initiate enrollment from course list  

---

## 2.3 Enroll in a Course

**User Story:** As a student, I want to enroll in courses so I can participate in them.

**Process Flow:**
1. Student views available courses  
2. Student selects a course to enroll  
3. Student clicks "Enroll" button  
4. System validates enrollment (no duplicates, etc.)  
5. System creates enrollment record  
6. System confirms enrollment  
7. Student sees confirmation message  

---

## 2.4 View My Enrolled Courses

**User Story:** As a student, I want to view all the courses I am enrolled in so I can track my studies.

**Process Flow:**
1. Student navigates to "My Enrollments" page  
2. System retrieves student's enrollments from API  
3. System displays enrolled courses  
4. Student can see all enrolled courses  
5. Student can view course details  
6. Student can deregister from courses  

---

## 2.5 Deregister from a Course

**User Story:** As a student, I want to deregister from a course so I can update my course load.

**Process Flow:**
1. Student views enrolled courses  
2. Student selects course to deregister  
3. Student clicks "Deregister" button  
4. System confirms action  
5. System removes enrollment record  
6. System confirms deregistration  
7. Student sees updated enrollment list  

---

# 3. Business Rules

## 3.1 Enrollment Rules

- A student can enroll in multiple courses  
- A course can have multiple students  
- Duplicate enrollments are prevented  
- Enrollment requires authentication  

## 3.2 Data Rules

- Student email must be unique  
- Course names must be unique (if applicable)  
- Enrollment records link student and course  

---

# 4. Process Dependencies

- Registration must occur before login  
- Login must occur before enrollment  
- Enrollment requires available courses  
- Deregistration requires existing enrollment
