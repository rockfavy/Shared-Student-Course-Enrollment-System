# 6. File Naming Conventions

- **C# files**: PascalCase (e.g., `EnrollmentContext.cs`)
- **Razor files**: PascalCase (e.g., `MyEnrollments.razor`)
- **Test files**: Suffix with `Tests` (e.g., `EnrollmentTests.cs`)
- **Documentation**: Kebab-case (e.g., `local-development-setup.md`)

---

# 7. Folder Organization Rules

- Group related files together
- Use feature-based organization for API
- Use component-based organization for Frontend
- Keep shared code in Shared project
- Separate tests by type (Unit, Integration, Functional)
