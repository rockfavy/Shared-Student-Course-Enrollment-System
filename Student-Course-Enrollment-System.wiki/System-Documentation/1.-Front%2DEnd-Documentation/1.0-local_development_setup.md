# Student Course Enrollment System – Local Environment Setup Guide  
---
# 1. Overview

Setup instructions for preparing, configuring, and running the Blazor WASM frontend locally. Follows the technical constraints, architecture, and module boundaries from the project requirements.

---

# 2. Workstation Requirements

## 2.1 Operating System
- Windows 10 or later  
- Administrator rights required for installation and certificate setup

## 2.2 Hardware Minimum
- 8 GB RAM (16 GB recommended)  
- Quad-core CPU  
- 20+ GB SSD space

## 3.1 Visual Studio 2022

Install **Visual Studio 2022 Community/Professional/Enterprise** with:

- ASP.NET and Web Development  
- .NET Desktop Development (optional helper tools)

### Recommended IDE Configuration

- Enable **Roslyn analyzers**  
- Enable **full solution analysis**

## 3.2 .NET SDK

Install the SDK that matches the project version (e.g., **.NET 8 LTS**).

Check installation:

```bash

```

---

## 3.3 Testing Tools

### bUnit (Blazor Component Testing)

bUnit is the primary testing framework for Blazor components. Install via NuGet:

```bash
dotnet add package bunit
dotnet add package bunit.web
```

Or add to test project `.csproj`:

```xml
<PackageReference Include="bunit" Version="1.34.7" />
<PackageReference Include="bunit.web" Version="1.34.7" />
```

bUnit enables:
- Component rendering tests
- Parameter binding validation
- Event handling tests
- State management verification
- Markup assertions

### xUnit (Test Framework)

xUnit is used as the test runner. Install via NuGet:

```bash
dotnet add package xunit
dotnet add package xunit.runner.visualstudio
```

# 4. API Configuration

The Blazor WASM frontend requires the API backend to be running.

## 4.1 Configure API URL

Add to `appsettings.Development.json`:

```json
{
  "BackendApiUrl": "https://localhost:7001"
}
```

## 4.2 Start API Backend

Ensure the Minimal API backend is running before starting the Blazor WASM application.

# 5. Repository Structure & Solution Opening

1. Clone the repository  
2. Open the `.sln` file in Visual Studio  
3. Set the Blazor WASM project as **Startup Project**  
4. Ensure build configuration = **Debug**  
5. Ensure environment = **Development**

# 6. Application Configuration

## 6.1 launchSettings.json

Ensure a development profile exists:

```json
{
  "profiles": {
    "StudentCourseEnrollment.Frontend": {
      "commandName": "Project",
      "launchBrowser": true,
      "environmentVariables": {
        "ASPNETCORE_ENVIRONMENT": "Development"
      },
      "applicationUrl": "https://localhost:5001"
    }
  }
}
```

## 6.2 Restore Dependencies

```bash
dotnet restore
```

(or allow Visual Studio to restore automatically)

## 6.3 Build & Run

CLI:

```bash
dotnet build  
dotnet run
```

Visual Studio:

- **F5** → Debug  
- **Ctrl + F5** → Run without debugging

# 7. Authentication Setup (Local Environment)

The application uses **OAuth 2.0 & OpenID Connect** for local development.

## 7.1 Local Authentication

Local environment uses OAuth 2.0 & OpenID Connect:

- OAuth 2.0 protocol for authorization
- OpenID Connect for authentication
- Token-based authentication flow
- Local identity provider configuration

## 7.2 Configure Authentication

Authentication is configured in `Program.cs`:

```csharp
builder.AddStudentCourseEnrollmentOpenIdConnect();
```

For local development, this configures OAuth 2.0 & OpenID Connect authentication.

## 7.3 Environment-Specific Authentication

| Environment | Authentication Provider |
|-------------|------------------------|
| Local | OAuth 2.0 & OpenID Connect |
| Stage | Keycloak |
| Production | Entra ID (Azure AD) |
